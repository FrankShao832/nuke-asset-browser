## 场景 2：整体界面 → 一目了然（30 秒）

**画面：** 浏览器窗口全屏展示，慢速从上到下扫过

**操作：** 
- 鼠标慢扫顶栏：用户头像 → 搜索框（高亮 300ms）
- 移到中央网格：6 张卡片一行，排列整齐
- 移到右侧边栏：Filter 按钮组高亮
- 移到底部状态栏：`all | latest | Showing 6 of 20 drafts | 🟢 PostgreSQL`

**旁白：**  
整个界面在 PySide6 上原生构建，没有 Web 技术，不额外起服务。  
顶栏有实时搜索框——300 毫秒防抖，输入即搜。  
中央是自定义 FlowLayout 缩略图网格，每行 6 张卡片。  
右侧是过滤面板，按作者、收藏、发布状态一键切换。  
底部状态栏告诉你当前数据量和后端状态——绿色就是 PostgreSQL 在线。

## 场景 3：搜索 + 过滤 —— 瞬间定位（40 秒）

**画面：** 聚焦搜索框和网格

**操作：**
1. 在搜索框键入 `grain` → 网格实时过滤，仅显示含 grain 标签的草稿
2. 清除搜索 → 全部恢复
3. 依次点击侧边栏：`My Drafts` → `Favorites` → `Published`
4. 切换排序 `Hottest`（按使用次数降序，最热排前）

**旁白：**  
搜索框输入 "grain"——名字、描述、标签，全部实时匹配。  
300 毫秒防抖，不会每按一个键就卡一下。  

侧边栏一键过滤：只看自己的草稿、只看收藏、只看已发布的正式工具。  
排序也在这里——最新创建排前面，或者按使用次数排，最热的置顶。

## 场景 4：卡片交互 —— 预览、收藏、右键（45 秒）

**画面：** 聚焦单张卡片

**操作：**
1. 鼠标悬停在一张卡片上 → 蓝色边框出现（hover 动效）
2. 点击收藏星 → 星变亮 → 状态栏 Toast 弹出 "Favorited: xxx"
3. 鼠标再点 → 取消收藏 → "Unfavorited: xxx"
4. 右键打开菜单：
   - 悬停 `Copy Path` → 点击
   - 悬停 `Delete` → 点击 → 弹出确认框 → 取消 → 再试一次 → 确认删除 → 网格更新

**旁白：**  
每张卡片悬停时有蓝色边框反馈。  
收藏星切换即时响应，底部会有 Toast 提示。  

右键菜单包含了日常最频繁的操作：收藏切换、复制文件路径——这对 Nuke 工作流特别有用，  
粘贴到 Read 节点或者脚本里就是完整路径。  
删除有二次确认，避免误删。

## 场景 5：缩略图展示 —— EXR、DPX、MOV、序列（60 秒）

**画面：** 展示不同类型草稿的缩略图

**操作：**
1. 从网格中找到一张 EXR 图片草稿 → 悬停 → 缩略图正常显示（色彩与 Nuke 一致）
2. 找到 DPX 草稿 → 悬停 → 同样正确的颜色
3. 找到 MOV 视频草稿 → 悬停 → 出现视频第一帧缩略图
4. 找到 Image Sequence 草稿 → 类型标签显示 🎞️ 绿色

**旁白：**  
Asset Browser 支持了生产中最常见的文件格式。

EXR 文件——用 OpenEXR 库原生解码，不做自动曝光，只用 gamma 2.2 校正，  
颜色和 Nuke Viewer 里看到的完全一致。  

DPX 文件——通过 ffmpeg 解码，不额外做色彩校正，所见即所得。  

MOV 视频——首帧作为缩略图，等会我们看它的悬停播放效果。  

Image Sequence——自动检测帧序列，不拆成一堆文件，整个序列作为一个草稿。

## 场景 6：重头戏 —— 悬停播放（45 秒）

**画面：** 聚焦序列和视频卡片

**操作：**
1. 鼠标移入一张 Image Sequence 卡片 → 缩略图开始 24fps 逐帧轮播 → 顺滑
2. 鼠标移出 → 恢复静态缩略图
3. 鼠标移入视频卡片 → 同样是动态预览 → 加载过程中显示 🎬 符号
4. 鼠标移出 → 停止

**旁白：**  
这是项目的核心交互亮点之一——悬停播放。

鼠标移入卡片，24fps 自动轮播序列帧，效果和你在 Nuke 里按播放键一样流畅。  
移出即停，不占资源。  

视频文件也一样——ffmpeg 全程均匀抽帧，首轮播放边播边缓存 PNG，  
第二次悬停就秒开。第一帧缺失？自动用上一帧填补，播放永不跳跃。

## 场景 7：拖拽导入 —— 文件夹、文件、进度反馈（50 秒）

**画面：** 从 Finder 拖拽到浏览器

**操作：**
1. 从 Finder 拖一个文件夹（内含 200 张 DPX 序列）到网格区域
2. 观察：状态栏出现 "Processing…" 进度条（居中蓝色条纹）
3. 网格刷新，出现一个新的Sequence草稿卡片（🎞️ 绿色标签）
4. 拖入一个 MOV 文件 → 进度条闪烁 → 新卡片出现（🎬）
5. 批量拖入 5 个独立的 .nk 文件 → 进度条显示 "Importing 3/5…"
6. 完成 → Toast "📥 Imported 5 drafts"

**旁白：**  
从 Finder 直接拖拽。

拖一个文件夹——系统自动扫描、检测帧序列，整个合成为一个草稿，不拆帧。  
拖单个文件——单张卡片。  
拖多个文件——进度条实时显示导入进度，状态栏全程提供反馈。

## 场景 8：Nuke 集成 —— 右键保存 + 拖入（45 秒）

**画面：** Nuke Node Graph + Asset Browser 联动

**操作：**
1. 在 Nuke 的 Node Graph 选中一个 Grade 节点
2. 右键 → `Save to Asset Browser` → 弹出保存对话框
3. 自动填充节点名 `Grade1` → 补上描述和标签
4. 点击 Save → Toast 提示 → 网格中出现新的 template 草稿
5. 反过来，从浏览器中拖一个模板 `.nk` 到 Nuke Node Graph → 节点创建成功

**旁白：**  
Nuke 集成是双向的。

在 Node Graph 选中节点，右键保存——  
自动填入节点名作为草稿名，你可以补上标签和描述。一个操作就把工具存进团队库。

反过来，从浏览器拖一个模板到 Node Graph——  
Nuke 原生导入，即时可用。整个流程不离开 Nuke 界面。

## 附录：录制提示

### 画面比例
- **1920×1080** 全屏录制
- Nuke 窗口占左侧 60%，浏览器占右侧 40%（或反之）
- 确保字体不缩小到看不清，必要时放大至 120%

### 鼠标设置
- 录制前把鼠标光标**调大**（macOS: 系统设置 → 辅助功能 → 显示 → 光标大小）
- 鼠标移动**慢而稳**，不要快速甩动

### 语音提示
- 语速中等偏慢，每分钟约 160-180 字
- 关键功能前**停顿半秒**，让观众看清画面变化
- "如"、"呃" 等口头禅宁可留静默也不要急

### 后期建议
- 每个场景之间加 **0.5 秒黑场** 或 **交叉淡入淡出**
