# Nuke Asset Browser v2.0.0 — 展示视频脚本（无声版）

> **总时长：** ~7 分钟  
> **场景数：** 15 个（11 个主场景 + 4 个技术彩蛋卡）  
> **风格：** 干净、节奏感、文字即旁白  
> **观众：** 合成师 / 管线 TD / 技术总监  
> **配乐：** 轻量电子/科技感 BGM（全程铺底，不抢注意力）  

---

## 核心原则

1. **无旁白** — 所有信息通过屏幕文字传达
2. **文字即旁白** — 每个场景的操作对应一句屏幕标注，简练、到位
3. **节奏放慢** — 比有声版多留 30% 时间给观众读文字
4. **文字动画** — 建议 fade-in 或打字机效果，与鼠标动作同步
5. **位置统一** — 主标注居下 1/3 区域，辅助信息居左上角

## 附录：无声版录制提示

### 文字标注布局（建议）

```
┌─────────────────────────────────┐
│                                 │
│                                 │  ← 主画面区域（操作发生的地方）
│                                 │
│                                 │
├─────────────────────────────────┤
│  [3]  搜索 "grain" → 实时匹配   │  ← 文字标注区（下 1/3）
│       300ms 防抖 · 不卡键       │     fade-in，与操作同步
└─────────────────────────────────┘
```

### 文字动画风格
- **入场：** fade-in（0.3s），不要打字机效果（太慢）
- **离场：** fade-out（0.2s）
- **切换：** 下一条文字直接从底部 fade-in 替换上一条
- **字体：** 白色 24-28px，无衬线（Helvetica / SF Pro / 微软雅黑）

### 技术彩蛋卡
- **风格：** 黑底 + 等宽字体灰白色文字
- **入场：** 快速缩放切入（0.2s），配合轻微音效（`pop` 或 `click`）
- **文字量：** 每卡不超过 6 行，每行不超过 20 个字

### 画面比例
- **1920×1080** 全屏录制
- 操作区域占满屏幕，必要时放大至 120%
- 文字标注区保持清晰，不要被操作遮挡

### 鼠标
- **光标调大**（macOS: 系统设置 → 辅助功能 → 显示 → 光标大小）
- 移动**慢而稳**，每个操作之间停顿 1-2 秒
- 点击时故意让光标停留 0.5 秒后操作，给观众反应时间
